Fundamentos Conceptuales

¿Cómo se Comunican R y JavaScript?

El Navegador (JavaScript)

  • Maneja la interfaz de usuario
  • Detecta eventos del usuario
  • Actualiza el DOM

El Servidor (R)

  • Ejecuta la lógica de negocio
  • Procesa datos
  • Genera outputs reactivos

WebSocket: Conexión bidireccional persistente que permite mensajes en tiempo real en ambas direcciones.

El Flujo de Comunicación

Dos Registros Centrales

Shiny mantiene dos registros para gestionar componentes:

// Registro de inputs
Shiny.inputBindings.register(myInputBinding, 'pkg.myInput');

// Registro de outputs
Shiny.outputBindings.register(myOutputBinding, 'pkg.myOutput');

Note

Cuando Shiny inicializa, llama a Shiny.bindAll() para escanear el DOM y asociar cada elemento con su binding correspondiente.

Anatomía de un Componente Personalizado

Lado R

  1. Función UI que genera HTML
  2. Dependencias CSS/JS
  3. Función render (outputs)
  4. Función update (inputs)

Lado JavaScript

  1. Clase que extiende el binding base
  2. Métodos requeridos
  3. Registro en Shiny

Input Binding: Métodos Esenciales

Método Propósito
find(scope) Localiza instancias del input en el DOM
getValue(el) Extrae el valor actual para enviar a R
subscribe(el, callback) Configura listeners de eventos
setValue(el, value) Establece el valor programáticamente
receiveMessage(el, data) Maneja mensajes desde R

Output Binding: Métodos Esenciales

Método Propósito
find(scope) Localiza instancias del output en el DOM
renderValue(el, data) Renderiza los datos recibidos de R
renderError(el, err) Muestra errores (opcional)
clearError(el) Limpia mensajes de error (opcional)

El Patrón es Consistente

Important

Una vez que entiendes el patrón de input binding (find → getValue → subscribe), puedes construir cualquier input.

Una vez que entiendes el patrón de output (find → renderValue), puedes renderizar cualquier cosa.

Paso 1: Implementar find()

El método find() localiza todas las instancias del input en el DOM.

find: function(scope) {
  return $(scope).find('.tri-state-input');
}

Tip

Usa selectores de clase (.clase) en lugar de ID para soportar múltiples instancias.

Paso 2: Implementar getValue()

El método getValue() extrae el valor actual del elemento.

getValue: function(el) {
  return $(el).data('selected');
}

Warning

Error común: Olvidar el return. Sin él, el binding falla silenciosamente y R recibe NULL.

Paso 3: Implementar subscribe()

El método subscribe() configura los event listeners.

subscribe: function(el, callback) {
  $(el).on('click.triStateBinding', '.tri-state-option', function(e) {
    var value = $(this).data('value');
    $(el).data('selected', value);

    // Actualizar estado visual
    $(el).find('.tri-state-option').removeClass('active');
    $(this).addClass('active');

    callback();
  });
}

Namespacing de Eventos

// ✅ Correcto: usa namespace
$(el).on('click.triStateBinding', handler);

// ❌ Incorrecto: sin namespace
$(el).on('click', handler);

Note

El namespace (.triStateBinding) permite que unsubscribe() remueva solo nuestros handlers sin afectar otros.

Paso 4: Implementar setValue() y receiveMessage()

Para que updateTriStateInput() funcione desde R:

setValue: function(el, value) {
  $(el).data('selected', value);
  $(el).find('.tri-state-option').removeClass('active');
  $(el).find('.tri-state-option[data-value="' + value + '"]').addClass('active');
},

receiveMessage: function(el, data) {
  if (data.hasOwnProperty('selected')) {
    this.setValue(el, data.selected);
    $(el).trigger('change');
  }
}

No Olvides el Evento Change

receiveMessage: function(el, data) {
  if (data.hasOwnProperty('selected')) {
    this.setValue(el, data.selected);
    $(el).trigger('change');  // ¡Importante!
  }
}

Warning

Sin $(el).trigger('change'), la actualización no se propaga a las dependencias reactivas.

También necesitamos escuchar ‘change’

subscribe: function(el, callback) {
  $(el).on('click.triStateBinding', '.tri-state-option', function(e) {
    // ... código de click ...
    callback();
  });

  // Para receiveMessage
  $(el).on('change.triStateBinding', function(e) {
    callback();
  });
}

Solución Completa del Input Binding

var triStateBinding = new Shiny.InputBinding();

$.extend(triStateBinding, {
  find: function(scope) {
    return $(scope).find('.tri-state-input');
  },

  getValue: function(el) {
    return $(el).data('selected');
  },

  subscribe: function(el, callback) {
    $(el).on('click.triStateBinding', '.tri-state-option', function(e) {
      var value = $(this).data('value');
      $(el).data('selected', value);
      $(el).find('.tri-state-option').removeClass('active');
      $(this).addClass('active');
      callback();
    });
    $(el).on('change.triStateBinding', function(e) {
      callback();
    });
  },

  setValue: function(el, value) {
    $(el).data('selected', value);
    $(el).find('.tri-state-option').removeClass('active');
    $(el).find('.tri-state-option[data-value="' + value + '"]').addClass('active');
  },

  receiveMessage: function(el, data) {
    if (data.hasOwnProperty('selected')) {
      this.setValue(el, data.selected);
      $(el).trigger('change');
    }
  }
});

Shiny.inputBindings.register(triStateBinding, 'workshop.triStateInput');

Extensión: Rate Policy

Si hay tiempo, añade limitación de frecuencia:

getRatePolicy: function() {
  return { policy: 'debounce', delay: 250 };
}

Políticas disponibles:

  • direct: Envía inmediatamente
  • debounce: Espera hasta que no haya nuevos valores
  • throttle: Máximo un envío por período

statCardOutput <- function(outputId, width = "200px") {
  tagList(
    statCardDependency(),
    tags$div(
      id = outputId,
      class = "stat-card",
      style = paste0("width: ", width, ";"),
      tags$div(class = "stat-title", "Loading..."),
      tags$div(class = "stat-value", "--")
    )
  )
}

La Función Render en R

renderStatCard <- function(expr) {
  func <- shiny::quoToFunction(rlang::enquo0(expr))

  shiny::createRenderFunction(
    func,
    transform = function(value, session, name, ...) {
      list(
        title = value$title,
        value = value$value,
        status = value$status

Paso 2: Implementar renderValue()

Este método recibe los datos de R y actualiza el DOM:

renderValue: function(el, data) {
  // Actualizar título y valor
  $(el).find('.stat-title').text(data.title);
  $(el).find('.stat-value').text(data.value);

  // Actualizar clase de estado
  $(el).removeClass('status-good status-warning status-bad');
  if (data.status) {
    $(el).addClass('status-' + data.status);
  }
}

Los Datos Llegan como JSON

  value: 75,
  status: "warning"

Recapitulación

Input Bindings

  1. find() - Localizar elementos
  2. getValue() - Extraer valor
  3. subscribe() - Escuchar eventos
  4. setValue() - Establecer valor
  5. receiveMessage() - Recibir de R

Output Bindings

  1. find() - Localizar elementos
  2. renderValue() - Renderizar datos
