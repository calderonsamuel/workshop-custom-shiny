Resources: Custom Shiny Inputs and Outputs

Quick Reference

Input Binding Methods

Method Required Purpose
find(scope) Yes Locate all instances of your input in the DOM
getValue(el) Yes Extract current value to send to R
subscribe(el, callback) Yes Set up event listeners for value changes
unsubscribe(el) No Clean up event listeners
setValue(el, value) No Programmatically set the value
receiveMessage(el, data) No Handle messages from R (for update functions)
getRatePolicy() No Control update frequency (debounce/throttle)
getType() No Specify custom R input handler
initialize(el) No One-time setup when binding is first attached

Output Binding Methods

Method Required Purpose
find(scope) Yes Locate all instances of your output in the DOM
renderValue(el, data) Yes Render data from R into the DOM element
renderError(el, error) No Display error messages
clearError(el) No Clear error display

Rate Policies

getRatePolicy: function() {
  return { policy: 'debounce', delay: 250 };
}
  • direct — Send immediately (no rate limiting)
  • debounce — Wait until no new values for delay ms (good for text inputs)
  • throttle — Send at most once per delay ms (good for sliders)

Debugging Tips

Enable WebSocket Tracing

options(shiny.trace = TRUE)

Shows all messages between browser and R server.

Enable Reactive Log

reactlog::reactlog_enable()
# Run your app, then:
shiny::reactlogShow()

Visualize the reactive dependency graph.

Browser Developer Tools

  • Console: Check for JavaScript errors
  • Network tab: Inspect WebSocket messages
  • Elements tab: Verify HTML structure matches your expectations

Use the debugger Statement

getValue: function(el) {
  debugger;  // Execution pauses here when dev tools are open
  return $(el).data('selected');
}

Common Patterns

Storing Configuration in HTML

Use a <script type="application/json"> tag to embed configuration:

# R side
tags$script(
  type = "application/json",
  `data-for` = inputId,
  jsonlite::toJSON(config, auto_unbox = TRUE)
)
// JavaScript side
var config = JSON.parse(
  $(el).find('script[data-for="' + el.id + '"]').html()
);

Programmatic Value Setting from JavaScript

Shiny.setInputValue("inputId", value);

// For events that should always trigger, even with same value:
Shiny.setInputValue("inputId", value, {priority: "event"});

Custom Input Handlers (R-side deserialization)

# In your package's .onLoad()
shiny::registerInputHandler("mypackage.dateTime", function(x, session, inputname) {
  as.POSIXct(x, origin = "1970-01-01")
})
// In JavaScript, use colon syntax
Shiny.setInputValue("timestamp:mypackage.dateTime", Date.now());

Custom Message Handlers

# R server
session$sendCustomMessage("myHandler", list(data = processed_data))
// JavaScript
Shiny.addCustomMessageHandler("myHandler", function(message) {
  console.log(message.data);
});
